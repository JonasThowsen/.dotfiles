# Ash Framework documentation references

Consult these sources when working on Ash tasks.

## Official docs

Start with current Ash documentation:
- https://ash-hq.org/
- https://hexdocs.pm/ash/

Use these for:
- resources
- actions
- relationships
- attributes
- identities
- validations
- changes
- calculations and aggregates
- policies and authorization
- code interfaces
- domains and framework patterns

## Related Ash packages

If the project uses related packages, consult their current docs as well, for example package docs on HexDocs for:
- ash_postgres
- ash_phoenix
- ash_authentication
- ash_admin
- cinder
- other Ash ecosystem libraries present in the project

## Local codebase check

After consulting official docs, inspect the local repository for:
- existing resources
- action naming and shapes
- policy patterns
- relationship modeling
- domain organization
- interface usage

Official docs provide the canonical framework patterns; the local codebase provides team-specific conventions.
